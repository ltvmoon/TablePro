# Filtering

The filter panel builds WHERE clauses without writing SQL. Pick columns, operators, and values visually, or switch to raw SQL mode for anything the builder can't express. Filters are per-tab, so each workspace keeps its own state.

## Filter Panel

Click the **Filter** button or press `Cmd+F` to open the filter panel. Each filter row specifies a column, operator, and value. Click **+** to add filters, use the AND/OR toggle to control logic, and click **Apply All** to activate selected filters. Click **Unset** to clear all filters.

## Operators

The filter builder supports 18 operators: equals, not equals, contains, starts with, ends with, greater than (>), greater or equal (>=), less than (<), less or equal (<=), IS NULL, IS NOT NULL, IS EMPTY, IS NOT EMPTY, IN, NOT IN, BETWEEN, and REGEX.

Operators without values (IS NULL, IS NOT NULL, IS EMPTY, IS NOT EMPTY) hide the value field.

### BETWEEN Operator

BETWEEN shows two value fields with an "and" label between them. Both values are required.

Enter comma-separated values to match against a list:

```
value1, value2, value3
```

## Raw SQL Mode

For conditions the visual builder can't express, switch to Raw SQL mode.

### Using Raw SQL

1. Click the column dropdown on a filter row
2. Select **Raw SQL** at the top of the list
3. The operator dropdown disappears and the value field becomes a free-text SQL input
4. Type your WHERE clause condition directly

**Examples:**

```sql

```

```sql
LOWER(name) LIKE '%smith%' AND status IN ('active', 'pending')
```

```sql
price * quantity > 1000
```

<Warning>
Raw SQL is injected directly into the WHERE clause. Make sure your syntax is correct for the connected database type.
</Warning>

## Quick Search

Quick Search filters across all visible columns at once, without building individual filter rows.

### Using Quick Search

1. Open the filter panel
2. Type in the **Quick search across all columns...** field
3. Press `Enter` to apply

## Filter Presets

Save your current filter configuration as a named preset for quick reuse.

### Saving a Preset

1. Build your filters in the filter panel
2. Click the **folder** icon in the filter header
3. Select **Save as Preset...**
4. Enter a name and click **Save**

### Managing Presets

| Action | How |
|--------|-----|
| Load | Click the preset name in the folder menu |
| Delete | Select **Delete Preset** > click the preset name |
| Overwrite | Save a new preset with the same name |

Presets are stored via UserDefaults and persist across sessions.

<Note>
Presets save the filter definitions (column, operator, value) but not whether they were applied. After loading a preset, click **Apply All** to activate the filters.
</Note>

## SQL Preview

See the exact WHERE clause that will be sent to the database before applying.

### Viewing the Preview

1. Build your filters
2. Click the **SQL** button in the filter footer
3. A sheet displays the generated WHERE clause

### What the Preview Shows

- The complete WHERE clause with proper quoting for your database type
- MySQL uses backticks (`` `column` ``)
- PostgreSQL uses double quotes (`"column"`)
- AND/OR logic based on the current mode
- A "Copy" button to copy the SQL to clipboard

If some (but not all) filters are selected, the preview shows only the selected filters. Otherwise, it shows all valid filters.

### Handling Invalid Filters

If filters exist but none are valid, the preview shows a message indicating what needs to be completed:

- Select a column
- Enter a value (if required by the operator)
- Fill in the second value for BETWEEN

## Per-Tab Filter State

Each tab maintains its own independent filter state, including the filter list, applied filters, and panel visibility. Switching between tabs preserves each tab's filters.

## Filter Settings

Click the **gear** icon to configure default column (Raw SQL, Primary Key, or any column), default operator (= or Contains), and panel behavior on open (Always Hide, Always Show, or Restore Last Filter).
